Windows 3.1x

Версии Windows 3.1x сделали Microsoft мировым лидером в области ОС для ПК. Рассказывают, что OS/2 в то время была более мощной и совершенной, но она полностью поддерживала приложения Windows 3.1x, и поэтому производители ПО писали свои приложения под Windows 3.1x, чтобы адресовать сразу обе платформы. Таким образом, если бы IBM отказалась от поддержки приложений Windows 3.1x, ей бы, возможно, удалось сохранить OS/2 на рынке. Но история рассудила иначе, и в начале 1990-х Windows 3.1x уже стояла на подавляющем большинстве ПК.

Windows 3.1

Windows 3.1, в отличие от Windows 3.0, не поддерживала реальный режим; интерфейс уже не менялся, единственно были изменены цвета по умолчанию.

Ctrl-Alt-Del

Начиная с Windows 3.1, нажатие комбинации Ctrl-Alt-Del не приводит к немедленной перезагрузке компьютера, а обрабатывается Windows как "запрос внимания". Если система не обнаруживает "не отвечающих" приложений, она просто предлагает вернуться обратно в Windows.
Если "не отвечающие" приложения находятся, то Windows предлагает принудительно закрыть их, и опять вернуться к нормальной работе.

Windows for Pen Computing

Windows for Pen Computing – версия Windows 3.1 для планшетных ПК, устанавливаемая поверх "обычной" Windows 3.1. Она позволяет управлять всеми функциями Windows, используя перо; для ввода текста используются экранная клавиатура и распознавание рукописного ввода.
На основе WfPC позже была создана ОС Windows CE.


Установка: приглашение
WfPC 1.0 – единственная существовавшая версия продукта, т.к. он потерпел полный коммерческий крах
В состав Windows SDK входил набор компонентов, позволявший тестировать WfPC и программы для неё на обычном ПК, используя вместо пера мышь. С непривычки это страшно неудобно, но достаточно для "получения представления" о продукте.
WfPC в этом случае работает в режиме "только перо", т.е. переключить мышь в "обычный" режим невозможно.

настройки пера: толщина и цвет проводимой линии, скорость реакции и т.д.
Любопытная возможность в WfPC – поворот дисплея, позволяющий работать в разрешении 480x640. Дисплей можно повернуть в любую сторону, но нельзя перевернуть "вверх ногами".

Блокнот

Здесь показаны две программы для ввода текста: Pen Palette для рукописного ввода и On-Screen Keyboard – экранная клавиатура. В Windows CE такие программы будут называться "метод ввода" (Input Method)
Две кнопки "с жёсткими углами" на панели инструментов Pen Palette на самом деле – меню Edit и Help.
Кнопки в среднем ряду вызывают Trainer и экранную клавиатуру. Таким образом, Pen Palette предоставляет доступ практически ко всем службам WfPC.
Можно выбрать три раскладки экранной клавиатуры: полную, только текстовые клавиши, только расширенные клавиши. Пока открыт диалог выбора и экранная клавиатура недоступна, соответствующая ей кнопка на панели инструментов Pen Palette засерена.
Вторая кнопка на панели Pen Palette переводит её в "отключенный" режим: поле для ввода текста спрятано, и окно занимает минимум места на экране. Также показано содержимое меню Edit – обратите внимание на необычные обозначения для жестов пера.
Демонстрационная программа Sample Form, входящая в состав WfPC, демонстрирует три элемента управления, ориентированные на работу с пером: "клеточки" для ввода букв, строка для ввода текста, поле для ввода многострочного текста. После того, как буква "написана" пером, через небольшое время она "распознаётся" – к сожалению, я не знаю, как этот процесс можно показать в динамике. Так текст вводится по одной букве, хотя можно и больше (если успеть – мне это удавалось редко).
Распознавание больших/маленьких букв не на высоте – эта проблема сохранилась и в Windows CE.

Текстовый редактор

Самая большая и сложная программа в составе WfPC – это Notebook, объединяющая многостраничную записную книжку и ежедневник. На "стартовой" странице можно редактировать иерархию страниц Notebook и переходить к ним; можно также проходить по ним последовательно с помощью элемента управления в левом нижнем углу. На каждой странице можно сочетать рукописные и текстовые заметки; рукописные заметки можно также распознавать и тем самым переводить в текст. В показанном меню есть возможность автоматического распознавания, когда рукописные буквы превращаются в текст немедленно; однако это преобразование можно затем отменить.
Вот во что превратился написанный мной ранее текст: даже отдельные буквы подчас распались на части. Размер текста автоматически подбирается под размер рукописных букв, поэтому неровный почерк сохраняется и в распознанном тексте.
Для распознавания рукописного текста используется кнопка "ab" на панели инструментов; кнопка с изображением извилистой линии отменяет распознавание и возвращает текст в том виде, как он был написан.
